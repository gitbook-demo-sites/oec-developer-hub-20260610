# OEC Developer Hub Demo

Source for a GitBook demo site showing how OEC could structure an external partner and developer portal with authenticated internal-only surfaces.

Each top-level folder is a separate GitBook space:

- `home` for the landing experience, portal model, and rollout framing
- `guides` for onboarding, implementation, testing, and launch guidance
- `api` for reference framing, auth, events, and operational API patterns
- `access` for login, permissions, internal overlays, and support operations
