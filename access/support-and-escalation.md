# Support and escalation

## External support path

- clear support entrypoint
- environment and product-family routing guidance
- required debugging details before a case is opened
- expected response model for launch-blocking issues

## Internal support path

- escalation owner by API family
- incident routing and severity guidance
- known issue overlays when partner-facing docs should stay stable

## Keep support close to the product

The header should expose support and login early so developers do not have to finish reading three pages before they know how to get help.
